ibmcloudsql.sql_magic¶

ibmcloudsql.sql_magic provides 2 classes

  • TimeSeriesTransformInput class which provides utilities for mapping user-friendly APIs to library-friendly APIs
  • TimeSeriesSchema class which provides hints about the schema of the data to support get_ts_datasource()
  • SQLMagic class which provides APIs to help constructing a complete SQL Query without knowing the details about IBM Cloud SQL-specific syntax

TimeSeriesTransformInput¶

A TimeSeriesTransformInput class that provides utilities for mapping from user-friendly time-series query into library-friendly time-series query

Example:

sql_stmt = """
ts_segment_by_time(ts, week, week)
"""
sql_stmt = """
ts_segment_by_time(ts, 604800000, 604800000)
"""
  • transform_sql(): the decorator that is applied on the SQLMagic.print_sql() and SQLMagic.format_() methods
  • ts_segment_by_time()

TimeSeriesSchema¶

  • unixtime_columns tells which columns contain time-stamp data and in unix time format

SQLMagic¶

A SQLMagic class is also a TimeSeriesSchema class

  • reset_() : to reset the internal storage of SQL statement (use this before constructing a new one)
  • print_sql(): to print and check the current content of SQL statement
  • with_() : provide table-name and the SQL query for that table
  • select_(): provide column names
  • from_table_(): the table name
  • from_cos_(): provide COS URL and format of data via format_type option
  • from_view_(): provide SQL statement that return a view
  • where_(): where condition
  • order_by_(): list of columns
  • group_by_(): list of columns
  • store_at_(): provide COS URL and format of data
  • partiton_objects_(): provide number of objects
  • partition_rows_(): provide number of rows per object
  • format_(): to apply transformation needed to map user-friendly Time-Series Queries into library-friendly Time-Series Queries

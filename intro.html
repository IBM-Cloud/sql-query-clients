Introduction¶

sql-query-client¶

Working with data on the cloud, in the form of a SQL Query, involves multiple parts.

    1. IBM Cloud access which control access to all IBM catalogs/resources

    1. IBM COS instance where data is stored. This includes access to data asset as part of a IBM Watson Studio project.

    1. IBM Cloud SQL Query instance where the SQL-based query service is provided

    1. IBM Watson Studio: which provides notebook environment for running notebook, in that (Python) code and data asset are also stored, implicitly, in the Project’s IBM COS instance.

    1. Visualization: can be done with Python code via IBM Watson Studio’s notebook

This sql-query-client library makes the interacting across these multiple parts possible.

The sql-query-client library comes with multiple submodules, each one links to another via subclass mechanism.

The submodules

  • utilities provides (A) - the IBM Cloud service connectivity-related functionality

  • cos provides (A, B) - the COS-related functionality

  • sql_query provides (A, B, C)

  • (D, E) is done via notebook-based examples of user-specific code which also utilizes above submodules

To install your package in “editable” mode

cd sql-query-client/Python
pip install -e .

To run the test on the package, run either

python -m pytest

pytest
